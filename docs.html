SevenRooms Booking API

A lightweight REST API for checking availability and booking tables at Dishoom restaurants. Purpose-built for voice agents — typical end-to-end latency under 1 second.

✓ Live 2 endpoints ~600ms avg

Overview

This API wraps the SevenRooms widget API and exposes two simple POST endpoints. No browser automation — just direct HTTP calls, making it fast enough for real-time voice agent use.

Authentication uses a static Bearer token. All requests and responses are JSON.

💡 The /book endpoint places a 5-minute hold then immediately confirms. Reservations are real — test with a real booking and cancel via the confirmation email.

Voice Agent Flow

1

Collect reservation intent

Agent gathers: venue, date, preferred time, party size

2

POST /check-availability

Returns bookable time slots near the requested time (~400ms)

3

Read back options to caller

"I have 19:00, 19:30 and 20:00 available — which works for you?"

4

Collect guest details

Agent gathers: first name, last name, email, phone number

5

POST /book

Pass the chosen slot's IDs + guest details. Returns confirmation (~500ms)

6

Confirm to caller

"You're booked! Confirmation number ABC123. See you at Dishoom!"

POST /check-availability Find available time slots

Returns bookable time slots for a given venue, date and party size. Optionally filter by preferred time (±2 hours).

Request body

FieldTypeRequiredDescription
venuestringrequiredVenue ID from /venues
datestringrequiredDate in YYYY-MM-DD format
party_sizeintegerrequiredNumber of guests
timestringoptionalPreferred time HH:MM — filters to ±2h window

Authorization: Bearer 85d6380def8534576e73c78f04aedf989d8b89b9d6bc94f9
Content-Type: application/json

  "party_size": 4,
  "time":       "19:00"  // optional — filters results to ±2h

⚠️ Important: The access_persistent_id and shift_persistent_id values are time-sensitive. Pass them directly to /book — don't cache them across sessions.

Error Responses

All errors return JSON with an error field and optional detail.

StatusMeaning
200Success
400Missing or invalid fields — check the error message
401Missing or wrong API key
409Slot no longer available — call /check-availability again
500Internal server error
502SevenRooms upstream error

Code Examples

const BASE = 'https://sevenrooms-api-production.up.railway.app';
const KEY  = '85d6380def8534576e73c78f04aedf989d8b89b9d6bc94f9';
const headers = {
  'Authorization': `Bearer ${KEY}`,
  'Content-Type':  'application/json',
};

// 1. Check availability
const avail = await fetch(`${BASE}/check-availability`, {
  method:  'POST',
  headers,
  body: JSON.stringify({
    venue:      'dishoom-manchester',
    date:       '2026-03-25',
    party_size: 4,
    time:       '19:00',
  }),
}).then(r => r.json());

const slot = avail.available_times[0];  // pick a slot

// 2. Book
const booking = await fetch(`${BASE}/book`, {
  method:  'POST',
  headers,
  body: JSON.stringify({
    venue:                'dishoom-manchester',
    date:                 '2026-03-25',
    time:                 slot.time,
    party_size:           4,
    access_persistent_id: slot.access_persistent_id,
    shift_persistent_id:  slot.shift_persistent_id,
    first_name:           'John',
    last_name:            'Smith',
    email:                'john@example.com',
    phone:                '+447700900000',
  }),
}).then(r => r.json());

console.log(booking.message);
